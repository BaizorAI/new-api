## 〇、与 plan20260715 方案 B 的差异对照

| 维度 | 方案 B (plan20260715) | 方案 C (本文档) |

## 一、模块定位与产品架构

### 1.1 产品定位

影视工作室（Film Studio）是 Baizor-API 平台的**独立创作工具模块**，与聊天、图像生成、一人公司并列。定位：

- **专业影视创作 pipeline**：从剧本到成片的全流程管理
- **AI 驱动**：每个阶段都有"快速生成"和"Agent 智能编排"双入口
- **结构化管理**：项目 → 阶段节点树 → 镜头/角色，不是纯对话

### 1.2 整体架构

```
┌─────────────────────────────────────────────────────────────────────┐
│                        Sidebar Product Rail                         │
│  概览 │ 工作台 │ 一人公司 │ 🎬 影视工作室 │ 团队协作 │ ...          │
│                                ↑ 独立顶层 Group                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  /studio                  → 项目列表（卡片网格）                       │
│  /studio/$projectId       → 项目看板（7 阶段节点 + 状态 + 依赖）      │
│  /studio/$projectId/$key  → 节点详情（数据表 + 嵌入式 Agent 对话）    │
│                                                                      │
├─────────────────────────────────────────────────────────────────────┤

│  ├─ Shots / Characters CRUD                                         │

```

```go
// model/studio_project.go
package model

import (
    "errors"
    "gorm.io/gorm"
)

const (
    StudioProjectStatusDraft      = 1
    StudioProjectStatusInProgress = 2
    StudioProjectStatusCompleted  = 3
    StudioProjectStatusArchived   = 4
)

type StudioProject struct {
    Id          int            `json:"id" gorm:"primaryKey;autoIncrement"`
    UserId      int            `json:"user_id" gorm:"index;not null"`
    TeamId      int            `json:"team_id" gorm:"index"`             // 0=个人项目
    Name        string         `json:"name" gorm:"type:varchar(128);not null"`
    Brief       string         `json:"brief" gorm:"type:text"`           // 创意简介
    Genre       string         `json:"genre" gorm:"type:varchar(32)"`    // short_drama, commercial, animation
    Status      int            `json:"status" gorm:"default:1"`          // 1=draft 2=in_progress 3=completed 4=archived
    StyleDNA    string         `json:"style_dna" gorm:"type:text"`       // JSON: 视觉风格参数
    CoverURL    string         `json:"cover_url" gorm:"type:varchar(500)"`
    CreatedAt   int64          `json:"created_at" gorm:"autoCreateTime:milli"`
    UpdatedAt   int64          `json:"updated_at" gorm:"autoUpdateTime:milli"`
    DeletedAt   gorm.DeletedAt `gorm:"index"`
}

func (p *StudioProject) Insert() error { return DB.Create(p).Error }

func (p *StudioProject) Update() error {
    return DB.Model(p).Select(
        "name", "brief", "genre", "status", "style_dna", "cover_url", "updated_at",
    ).Updates(p).Error
}

func (p *StudioProject) Delete() error { return DB.Delete(p).Error }

func GetStudioProjectById(id int) (*StudioProject, error) {
    if id == 0 {
        return nil, errors.New("id is required")
    }
    var p StudioProject
    err := DB.First(&p, "id = ?", id).Error
    return &p, err
}

func GetAllStudioProjects(userId int, status int, startIdx, num int) ([]*StudioProject, int64, error) {
    tx := DB.Model(&StudioProject{})
    if userId > 0 {
        tx = tx.Where("user_id = ?", userId)
    }
    if status > 0 {
        tx = tx.Where("status = ?", status)
    }
    var total int64
    if err := tx.Count(&total).Error; err != nil {
        return nil, 0, err
    }
    var projects []*StudioProject
    err := tx.Order("id desc").Limit(num).Offset(startIdx).Find(&projects).Error
    return projects, total, err
}
```

### 3.2 StudioStage — 阶段节点

```go
// model/studio_stage.go
package model

const (
    StageStatusNotStarted    = 0
    StageStatusInProgress    = 1
    StageStatusCompleted     = 2
    StageStatusNeedsRevision = 3
)

type StudioStage struct {
    Id          int    `json:"id" gorm:"primaryKey;autoIncrement"`
    ProjectId   int    `json:"project_id" gorm:"index;uniqueIndex:idx_project_key"`
    Key         string `json:"key" gorm:"type:varchar(32);uniqueIndex:idx_project_key"`
    Name        string `json:"name" gorm:"type:varchar(64)"`
    Order       int    `json:"order"`
    Status      int    `json:"status" gorm:"default:0"`                    // 0=未开始 1=进行中 2=已完成 3=需修改
    AutoSkill   string `json:"auto_skill" gorm:"type:varchar(64)"`        // 关联 Hermes Skill
    TotalItems  int    `json:"total_items" gorm:"default:0"`
    DoneItems   int    `json:"done_items" gorm:"default:0"`
    OutputData  string `json:"output_data" gorm:"type:text"`              // JSON: 阶段结构化产出
    CreatedAt   int64  `json:"created_at" gorm:"autoCreateTime:milli"`
    UpdatedAt   int64  `json:"updated_at" gorm:"autoUpdateTime:milli"`
}

// 创建项目时自动生成的 7 个阶段定义
var DefaultProjectStages = []struct {
    Key       string
    Name      string
    Order     int
    AutoSkill string
}{
    {"script",     "剧本",     1, "script-analyzer"},
    {"characters", "角色",     2, "character-designer"},
    {"storyboard", "分镜",     3, "shot-planner"},
    {"image_gen",  "图像生成", 4, "batch-generator"},
    {"video_gen",  "视频生成", 5, "batch-generator"},
    {"post",       "后期合成", 6, "post-production"},
    {"review",     "审核发布", 7, ""},
}

// CreateDefaultStages 在项目创建后调用，自动生成 7 个阶段节点
func CreateDefaultStages(projectId int) error {
    stages := make([]StudioStage, len(DefaultProjectStages))
    for i, s := range DefaultProjectStages {
        stages[i] = StudioStage{
            ProjectId: projectId,
            Key:       s.Key,
            Name:      s.Name,
            Order:     s.Order,
            AutoSkill: s.AutoSkill,
        }
    }
    return DB.Create(&stages).Error
}

func GetStudioStagesByProjectId(projectId int) ([]StudioStage, error) {
    var stages []StudioStage
    err := DB.Where("project_id = ?", projectId).Order("`order` asc").Find(&stages).Error
    return stages, err
}

func UpdateStudioStage(stage *StudioStage) error {
    return DB.Model(stage).Select(
        "status", "total_items", "done_items", "output_data", "updated_at",
    ).Updates(stage).Error
}

func GetStudioStageByProjectAndKey(projectId int, key string) (*StudioStage, error) {
    var stage StudioStage
    err := DB.Where("project_id = ? AND `key` = ?", projectId, key).First(&stage).Error
    return &stage, err
}
```

### 4.2 API 详细说明

#### 项目创建（核心：自动生成节点树）

```
POST /api/studio/projects
Body: {
    "name": "都市恋爱短剧",
    "brief": "讲述都市青年的爱情故事...",
    "genre": "short_drama",           // short_drama | commercial | animation | music_video
    "style_dna": "{\"style\":\"cinematic\",\"palette\":\"warm\"}"
}

Response 201:
{
    "success": true,
    "data": {
        "id": 1,
        "name": "都市恋爱短剧",
        "status": 1,
        "stages": [
            {"key":"script","name":"剧本","order":1,"status":0,"auto_skill":"script-analyzer"},
            {"key":"characters","name":"角色","order":2,"status":0,"auto_skill":"character-designer"},
            {"key":"storyboard","name":"分镜","order":3,"status":0,"auto_skill":"shot-planner"},
            {"key":"image_gen","name":"图像生成","order":4,"status":0,"auto_skill":"batch-generator"},
            {"key":"video_gen","name":"视频生成","order":5,"status":0,"auto_skill":"batch-generator"},
            {"key":"post","name":"后期合成","order":6,"status":0,"auto_skill":"post-production"},
            {"key":"review","name":"审核发布","order":7,"status":0,"auto_skill":""}
        ]
    }
}
```

**关键**：`CreateStudioProject` handler 在 `DB.Create(&project)` 成功后，立即调用 `model.CreateDefaultStages(project.Id)` 自动生成 7 个阶段节点。

#### 快速生成

```
POST /api/studio/projects/:id/quick-generate
Body: {
    "type": "image" | "video" | "i2v" | "describe" | "analyze",
    "prompt": "咖啡厅全景，暖色调，电影感",
    "stage_key": "image_gen",
    "shot_id": 5,            // 可选，关联到具体镜头
    "model": "huayu-drama-4",
    "size": "1024x1024"
}
```

分发逻辑：
| type | 调用方式 | 返回 |
|------|---------|------|
| `image` | Go 直调 relay `/pg/images/generations` | 同步返回图片 URL |
| `video` | 创建 HermesExecutionTask → 异步调 `/v1/videos/generations` | 返回 task_id |
| `i2v` | 创建 Task → 异步调 `/v1/videos/image-to-video` | 返回 task_id |
| `describe` | Go 直调 `/v1/chat/completions`（VL 模型） | 同步返回描述文本 |
| `analyze` | Go 直调 `/v1/chat/completions`（structured output） | 同步返回结构化 JSON |

#### Agent 创作

```
POST /api/studio/projects/:id/agent-create
Body: {
    "skill": "script-analyzer" | "character-designer" | "shot-planner" | "batch-generator" | "post-production" | "full-pipeline",
    "stage_key": "script",
    "context": "..."          // 可选，传给 Agent 的上下文
}

Response 202:
{
    "success": true,
    "data": {
        "task_id": "studio_abc123",
        "status": "queued"
    }
}
```

## 七、阶段依赖图与状态流转

### 7.1 依赖图

```
📝 script ──────┬──→ 🎭 characters
                │
                └──→ 🎬 storyboard ──→ 🖼️ image_gen ──→ 🎥 video_gen ──→ ✂️ post ──→ ✅ review
```

未满足依赖的阶段显示为灰色/禁用，提示"需先完成 XX"。

### 7.2 状态流转

```
⬜ 未开始 (0) ──→ 🔄 进行中 (1) ──→ ✅ 已完成 (2)
                       │
                       ↓
                 ❌ 需修改 (3) ──→ 🔄 进行中 (1)
```

### 7.3 状态自动计算规则

| 阶段 | 自动计算条件 |
|------|-------------|
| 剧本 | `output_data` 非空 → ✅ |
| 角色 | 所有角色有 `reference_url` → ✅；部分有 → 🔄 (done/total) |
| 分镜 | 所有镜头创建完成 → ✅；部分 → 🔄 |
| 图像生成 | 所有镜头有 `image_url` → ✅；部分 → 🔄 (done/total) |
| 视频生成 | 所有镜头有 `video_url` → ✅；部分 → 🔄 |
| 后期合成 | `output_data` 含最终视频 URL → ✅ |
| 审核发布 | 用户手动切换 |

状态由后端 `UpdateStudioStage` 更新 `total_items` / `done_items`，前端根据 `done_items/total_items` 比值显示进度。

- `X-Baizor-Hermes-Session: <session_id>`
- `X-Baizor-Hermes-Workspace: studio_project_{id}_stage_{key}`

### 9.2 每个阶段的 Skill 和 System Prompt

| 阶段 | Skill | System Prompt 要点 |
|------|-------|--------------------|
| 剧本 | `script-analyzer` | 分析剧本，输出结构化场景/镜头/角色 JSON |
| 角色 | `character-designer` | 设计角色卡，生成参考图 prompt，输出一致性参数 |
| 分镜 | `shot-planner` | 规划镜头景别/运镜/时长，输出镜头列表 |
| 图像生成 | `batch-generator` | 批量生成关键帧图像，质量检查，失败重试 |
| 视频生成 | `batch-generator` | 批量 i2v 或 t2v，管理异步任务，监控进度 |
| 后期合成 | `post-production` | 编排 FFmpeg 拼接视频，添加转场/字幕/音效 |
| 审核发布 | — | 无 Agent（人工审核） |

### 9.3 Skill 文件结构

```
hermes-agent/skills/magicalbrush/
  ├─ script-analyzer.md        剧本 → 结构化场景/镜头/角色列表
  ├─ character-designer.md     角色 → 参考图 prompt + 一致性参数
  ├─ shot-planner.md           场景 → 镜头规划 + 景别/运镜选择
  ├─ batch-generator.md        批量镜头生成编排（图→视频 pipeline）
  ├─ consistency-guard.md      多镜头角色一致性校验 + 修复
  ├─ quality-checker.md        质量评分 + 异常诊断 + 修复建议
  ├─ post-production.md        素材 → FFmpeg 后期编排
  └─ full-pipeline.md          端到端：拆解→规划→生图→生视频→合成
```
