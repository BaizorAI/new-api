# 白泽 AI 平台投产前产品需求文档（PRD）

- **文档版本**：2026.07.21
- **适用项目**：New-API + HermesAgent SideCar
- **目标阶段**：投产前 P1 加固与可运维收尾
- **文档状态**：草案，待评审后冻结为投产基线

> 归属说明：底层工程能力来自 New-API 项目体系，项目身份、署名、版权及 QuantumNous 等受保护信息完整保留，不删除、替换或弱化。

---

## 1. 产品定位

白泽 AI 平台（Baizor）是基于 New-API 构建的 AI 中台（AI Hub），通过统一网关聚合 40+ 上游模型提供商（OpenAI、Claude、Gemini、Azure、DeepSeek、Qwen 等），为组织提供：

- 统一的 OpenAI-compatible API 出口
- 用户、团队、令牌、额度、模型权限治理
- 多租户计费、审计日志、安全策略
- 可选的 HermesAgent SideCar 扩展（技能中心、工具集、消息平台、微信扫码）

平台同时支持在线使用与私有化部署，面向政企、教育、科研、产业园区等需要数据不出域、信创适配和统一治理的场景。

---

## 2. 目标

### 2.1 业务目标

1. 在私有化/生产环境中稳定运行 New-API 核心网关 + HermesAgent SideCar。
2. 提供可审计、可回滚、可监控的完整部署方案。
3. 补齐投产前必须的安全与可运维短板，达到生产上线最低标准。

### 2.2 技术目标

1. 所有默认凭据从 compose 文件解耦到 `.env`，并强制使用强随机密钥。
2. 核心服务具备 CPU/内存资源限制与日志轮转。
3. 数据库与关键业务数据具备备份/恢复能力。
4. 关键服务具备健康检查与基本可观测性。
5. Hermes SideCar 与 New-API 之间的认证、代理、回调链路安全可控。

## 6. 架构概要

### 6.1 后端

- **语言/框架**：Go 1.22+，Gin，GORM v2
- **数据库**：SQLite / MySQL ≥ 5.7.8 / PostgreSQL ≥ 9.6
- **缓存**：Redis（go-redis）+ 内存缓存
- **前端嵌入**：`web/default/dist` 通过 Gin static 提供
- **主要模块**：
  - `router/`：HTTP 路由定义
  - `controller/`：请求处理
  - `service/`：业务逻辑
  - `model/`：数据模型与 DB 访问
  - `relay/`：上游模型适配与统一转发
  - `middleware/`：认证、限流、分发、审计

### 6.2 前端

- **技术栈**：React 19、TypeScript、Rsbuild、Base UI、Tailwind CSS 4
- **多语言**：i18next，支持 en/zh/fr/ru/ja/vi
- **Playground**：Chat、Image、Video、HermesAgent

### 6.3 HermesAgent SideCar

- **容器**：`baizor-hermes`
- **暴露方式**：仅 Docker 内部网络 `http://baizor-hermes:8642/v1`
- **调用链**：
  1. 用户 / 业务系统 → New-API API Key
  2. New-API 进行身份、团队、额度、模型权限校验
  3. 通过 `/pg/hermes/*` 代理到 Hermes
  4. Hermes 需要模型时回调 `http://new-api:3000/v1`
- **隔离**：按 `X-Hermes-User-Id` 落盘到 `/opt/data/baizor-users/<userId>/`

## 9. 部署与运维需求

### 9.1 环境变量规范

生产 `.env` 至少包含：

```dotenv
# 平台密钥
SESSION_SECRET=<>=64 chars random
CRYPTO_SECRET=<>=64 chars random

# 数据库
POSTGRES_USER=root
POSTGRES_PASSWORD=<>=32 chars random
POSTGRES_DB=new-api

# Redis
REDIS_PASSWORD=<>=32 chars random

# Hermes SideCar
HERMES_SIDECAR_ENABLED=true
HERMES_IMAGE=ccr.ccs.tencentyun.com/lucky/baizor-hermes:1.0.27
HERMES_API_SERVER_URL=http://baizor-hermes:8642
HERMES_API_SERVER_KEY=<>=64 chars random
HERMES_API_SERVER_PORT=8642
HERMES_INFERENCE_BASE_URL=http://new-api:3000/v1
HERMES_INFERENCE_API_KEY=<platform token for hermes>
HERMES_WEIXIN_QR_ENABLED=true
```

### 9.2 启动命令

本地镜像环境：

```bash
docker compose -f docker-compose.yml -f docker-compose.local.yml -f docker-compose.hermes.yml up -d
```

生产固定 tag 环境：

```bash
export NEW_API_IMAGE=ccr.ccs.tencentyun.com/lucky/baizor-newapi:1.4.56
export HERMES_IMAGE=ccr.ccs.tencentyun.com/lucky/baizor-hermes:1.0.27
docker compose -f docker-compose.yml -f docker-compose.hermes.yml up -d
```

### 9.3 备份需求

1. PostgreSQL：每日 `pg_dump` 到对象存储或异地目录，保留 7-30 天。
2. `hermes-data`：定时打包 `/opt/data` 目录。
3. `data/`、`logs/`、`image-playground/`、`video-playground/`：按业务重要性选择备份策略。
4. 提供一键恢复脚本，支持从最新备份重建容器与数据卷。

### 9.4 发布流程

1. 在测试环境验证镜像 tag 与 `.env`。
2. 拉取/构建固定 tag 镜像。
3. 执行 `docker compose pull && docker compose up -d`。
4. 验证 `/api/status`、Hermes `/health`、关键 Playground 功能。
5. 保留回滚命令：切回上一 tag 并重启。

## 12. 风险与假设

| 风险 | 影响 | 缓解措施 |
|---|---|---|
| 宿主机资源不足（尤其 Hermes 内存） | Hermes OOM 或推理失败 | 根据负载调整 `memory` 限制，预留 2G swap |
| `.env` 泄露 | 全部密钥与数据暴露 | 文件权限 600，不提交 Git，使用 secret 管理 |
| 未备份导致数据丢失 | 无法恢复用户/额度/审计 | 每日自动备份 + 异地存储 |
| TLS 证书过期 | 外部服务中断 | 配置自动续期（acme.sh / certbot） |
| Hermes 递归调用 | 无限循环导致额度与资源耗尽 | 服务账号仅授权底层模型，禁止调用 `hermes-agent` |
