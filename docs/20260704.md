 主要功能变动

  - API Server 增强：新增 model_routes，同一个 API Server 可以按请求里的 model alias 路由到不同 provider/model；GET /v1/models 会列出这些 alias。
  - API Server 安全更严：API_SERVER_KEY 现在启动时会强校验，空值、占位值、过短 key 可能导致 API server 拒绝启动。
  - Gateway 会话/搜索增强：新增 /sessions search <query>，SessionDB 增加搜索能力、handoff 索引、会话元数据补全逻辑。
  - Gateway 安全和稳定性增强：审批回复支持普通文本 yes/no/approve/deny；重启/恢复流程加了循环保护；gateway 图片、媒体路径、.env 文件访问、浏览器私页访问
    等安全边界更严格。

  - 模型/Provider 行为变动：vision、image-gen、TTS、MOA、自定义 provider、extra headers、fallback provider 等都有修复；自定义 provider slug 和 session
    model override 更稳定。

  - Dashboard/Web UI 变动：新增 Hermes Console UI/WebSocket、Skills Hub、MCP 管理页、Capabilities 页面整合、更多桌面端/网页端 UI 重构。
  - 工具系统变动：browser、approval、delegate、file、web_tools、skill_manager、video/xAI 等大量工具修复和安全 hardening。
  - 依赖和版本变动：pyproject.toml 版本从 0.17.0 到 0.18.0；新增/更新 Python 和 Node 依赖，包括显式 pin cryptography==46.0.7，Node 侧新增 @xterm/addon-
    serialize、d3-force、fflate 等。

  重新部署的影响

  - 建议做完整重建，不要只替换代码后热重启。Dockerfile 变了，新增 apps/shared/ copy；uv.lock、package-lock.json 都变了，需要重新安装/同步依赖。
  - 如果部署环境使用 API Server，确认 API_SERVER_KEY 是强随机值，长度至少 16，最好 openssl rand -hex 32 级别。否则新版本可能启动失败。
  - 如果某些平台配置了 dm_policy/group_policy: open，现在必须显式开启 GATEWAY_ALLOW_ALL_USERS=true 或对应平台 allow-all env，否则 gateway 会拒绝启动，避
    免误开放。

  - state.db 会走 schema/index 变化。一般应自动处理，但部署前建议备份 Hermes home/state 数据，尤其是 ~/.hermes/state.db、sessions、config。
  - Dashboard/网页端构建产物会变，重新部署后 UI 行为会明显不同，尤其是 Console、Capabilities、Skills/MCP 页面。
  - 安全策略更严后，之前依赖读取 .env、本地敏感文件、私页浏览器内容、任意媒体路径的工作流可能会被拒绝，这是预期安全收紧。
  - 你的本地 Baizor 功能我在冲突里保留了：API request skills、Baizor attribution headers、request work dir、Weixin scoped runtime fallback 等没有被上游覆
    盖。
