# MagicalBrush 技术实施方案 — 基于 Baizor-API + Hermes-Agent

> 日期：2026-07-08
> 来源：安妮天算-MagicalBrush 商业计划书
> 定位：面向影视/动漫/广告的企业级 AIGC 全栈生产平台

---

## 一、总体架构

```
Web 工作台
  ├─ 项目 / 角色 / 场景 / 分镜 / 审批          ← Baizor-API 前端扩展
  ├─ Hermes Agent（脚本拆解、镜头规划、工作流选择、异常解释）  ← Hermes 引擎 + 领域 Skill
  ├─ Skill Registry（SKILL.md + 参数 JSON Schema + 版本）    ← Hermes 已有
  ├─ 队列与调度（任务、GPU、优先级、成本、重试）              ← Baizor-API Task 系统复用
  ├─ SeeDance 2.0 接口（图像、视频、音频、LoRA）             ← Baizor-API 新增 Channel Adapter
  ├─ 资产库（对象存储 + 元数据 + 向量检索）                   ← 需新建
  ├─ 后期服务（FFmpeg、字幕、配音、工程导出）                 ← Hermes Skill 编排
  └─ 审计/确权（哈希、时间戳、输入授权、日志、审批流）         ← Baizor-API 审计扩展
```

### 系统分工

| 系统 | 角色 | 已有能力 |
|------|------|---------|
| **Baizor-API** (Go) | 网关 + 调度 + 计费 + 前端 + 数据层 | 40+ AI 供应商适配、BillingSession 计费、Task 异步队列、RBAC/Team、React 管理台 |
| **Hermes-Agent** (Python) | AI 大脑 + 工具执行 + 工作流编排 | 29+ LLM 供应商、109+ 工具、Skill 注册/生命周期、子 Agent 委派、看板协作、MCP 客户端/服务端 |

两者现有集成点：
- Hermes 通过 Baizor-API 的 `/v1/chat/completions` 调用 LLM（复用路由、计费、重试）
- Baizor-API 有 `HermesExecutionTask` 模型管理 Agent 长任务
- 前端已有 `/hermes-playground/` 页面

## 三、总体可行性评估

```
模块                    复用度    新建量    可行性    主要工作
──────────────────────────────────────────────────────────────
Web 工作台              70%      中等      🟢      领域模型 + Studio 路由组 + 分镜 Canvas UI
Hermes Agent           95%      低        🟢      5-8 个 SKILL.md 文件
Skill Registry         90%      低        🟢      加 version/category 字段
队列与调度             85%      低        🟢      Task 加 Priority/GpuType 字段 + 新 Channel Type
SeeDance 2.0 接口      75%      中等      🟢      新 Channel Adapter（参考 Kling）
资产库                 30%      中高      🟡      元数据模型 + 对象存储抽象 + 向量检索
后期服务               50%      中等      🟡      FFmpeg Skill 集合 + EDL 导出
审计/确权              60%      中等      🟡      哈希服务 + 版权链对接 + 审批流
──────────────────────────────────────────────────────────────
整体                   ~65%     中等      🟢 可行
```

### 结论

**方案可行，且复用度显著高于从零搭建。**

核心优势：
- **AI 网关不用重写**：Baizor-API 40+ 供应商接入、BillingSession 计费、通道路由、任务队列全部复用
- **Agent 引擎不用重写**：Hermes 109+ 工具、Skill 注册、子 Agent 编排、记忆系统直接用
- **用户体系不用重写**：RBAC、Team、支付、订阅全部复用
- **视频/图像生成管线不用重写**：已有 7+ 视频供应商适配器，SeeDance 只需按模式新增

最大新建工作量集中在三块：
1. **领域数据模型**（项目/角色/场景/分镜/审批 — ~6 个 Go 文件）
2. **资产库**（对象存储抽象 + 元数据 + 向量检索 — ~8 个 Go 文件）
3. **分镜编辑器 UI**（Canvas 前端组件 — 前端工作量最大的单项）

## 五、关键技术决策

| 决策点 | 推荐方案 | 备选 | 理由 |
|--------|---------|------|------|
| 分镜编辑器 | Konva.js | Fabric.js | 更轻量，React 集成更好 |
| 对象存储 | MinIO (私有化) + 阿里云 OSS (SaaS) | AWS S3 | MinIO S3 兼容，私有化零成本 |
| 向量检索 | 一期内存暴力搜 → 二期 pgvector | Milvus | 一期资产量 <10 万无需独立向量库 |
| 工程导出 | 一期 EDL → 二期 FCP XML | DaVinci XML | EDL 最通用，实现简单 |
| 版权存证 | 安妮版权链 | 蚂蚁链 | 与甲方（安妮天算）技术栈对齐 |
| LoRA 训练 | 作为 Task 异步任务 | 独立训练服务 | 复用现有 Task 基础设施 |
