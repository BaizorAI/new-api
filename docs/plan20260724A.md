# ComfyUI Playground Canvas Enhancement Plan

Date: 2026-07-24

## Overview

Enable full canvas interactivity in the ComfyUI Playground. Four features in one pass:
1. **Node dragging** — reposition nodes freely with persistent positions
2. **Node connecting** — full handle architecture upgrade (per-field target handles, per-output source handles), drag-to-connect
3. **Add/delete nodes** — static node library panel, Delete key support
4. **I2V workflow** — automatically available via workflow selector (no code changes needed)

---

## Files to Modify (in order)

### Step 1: `types.ts`
Add two new interfaces:
```typescript
export interface NodePositions {
  [nodeId: string]: { x: number; y: number }
}
export interface NodeLibraryEntry {
  classType: string
  category: 'loaders' | 'conditioning' | 'latent' | 'sampling' | 'output' | 'primitives' | 'other'
  defaultInputs: Record<string, ComfyuiNodeInput>
  defaultMeta?: { title: string }
  outputCount: number
  searchAliases?: string[]
}
```

### Step 2: `node-library.ts` (NEW)
Static TypeScript list of available ComfyUI node types covering all categories:
- **Loaders**: CheckpointLoaderSimple, UNETLoader, CLIPLoader, VAELoader, LoadImage, LoadVideo
- **Conditioning**: CLIPTextEncode, CLIPTextEncodeSDXL
- **Latent**: EmptyLatentImage
- **Sampling**: KSampler, SamplerCustomAdvanced, BasicScheduler, LTXVScheduler, RandomNoise
- **Output**: VAEDecode, VHS_VideoCombine, SaveImage, PreviewImage
- **Primitives**: PrimitiveInt, PrimitiveFloat, PrimitiveStringMultiline
- Each entry has `outputCount`, `defaultInputs`, `searchAliases`
- Exports `filterNodeLibrary(query, category)` function

### Step 3: `workflow-parser.ts` — Core Logic
**A. Add `getNodeOutputCount(classType)` helper** — looks up library entry; fallback: 0 for sinks, 1 for everything else

**B. Update `layoutNodes(wf, savedPositions?)`** — apply saved positions first, then auto-layout remaining nodes in grid; avoid overwriting saved positions

**C. Update `parseWorkflowToGraph(wf, onInputChange, savedPositions?, onDeleteNode?)`** — accept new params; build `targetHandles: string[]` (one per input field) and `sourceHandles: number[]` (one per output) in node data; include edge metadata (`targetNodeId`, `targetField`, `sourceNodeId`, `sourceOutput`) on each edge

**D. Remove old booleans** — `hasConnections`, `isSource`, `hasTargetHandles` no longer needed

**E. New pure functions:**
- `addNodeToWorkflow(wf, nodeId, classType, defaultInputs, meta?)` — returns new workflow with added node
- `removeNodeFromWorkflow(wf, nodeId)` — removes node AND cleans all connections referencing it from other nodes
- `addConnectionToWorkflow(wf, sourceId, sourceOutput, targetId, targetField)` — set `[sourceId, sourceOutput]` on target input
- `removeConnectionFromWorkflow(wf, nodeId, fieldName)` — reset connection to `''`

### Step 4: `comfyui-node.tsx` — Handle Architecture Upgrade
**Update `ComfyuiNodeData`:**
- Remove: `hasConnections`, `isSource`, `hasTargetHandles`
- Add: `targetHandles: string[]`, `sourceHandles: number[]`, `onDelete?: () => void`

**Handle rendering:**
- Target handles (left): one `<Handle id={fieldName} type="target">` per input field, stacked vertically
- Source handles (right): one `<Handle id={String(idx)} type="source">` per output index
- Handle IDs MUST match `edge.targetHandle`/`edge.sourceHandle` for drag-to-connect to work

**Delete button:** small X in top-right corner of node header, visible on hover (`group-hover:opacity-100`); calls `data.onDelete`

### Step 5: `workflow-canvas.tsx` — Enable Interactivity
**New props:**
```typescript
savedPositions: NodePositions
onPositionsChange: (positions: NodePositions) => void
onNodeDragComplete: () => void  // trigger undo push
onViewportCenterChange: (center: {x: number, y: number}) => void
```

**New handlers:**
- `handleNodeDragStop` — capture position, call onPositionsChange + onNodeDragComplete
- `handleConnect` — validate connection, call addConnectionToWorkflow, pass to onWorkflowChange
- `handleEdgesDelete` — parse edge data, call removeConnectionFromWorkflow, pass to onWorkflowChange
- Track viewport center via `onMove` for node placement

**ReactFlow props change:**
```tsx
nodesDraggable={true}        // was false
nodesConnectable={true}      // was false
onNodeDragStop={handleNodeDragStop}
onConnect={handleConnect}
onEdgesDelete={handleEdgesDelete}
```

### Step 6: `node-library-panel.tsx` (NEW)

### Step 7: `index.tsx` — Wire Everything Together

**Keyboard shortcuts** — add to existing `useEffect`:

**Pass new props** to WorkflowCanvas and SidebarPanel

## Undo/Redo Integration

| Operation | Push to undo? | How |
|-----------|--------------|-----|
| Node drag | No | Positions are visual-only, not part of workflow data |
| Connect/disconnect | Yes | Via handleWorkflowChange → pushWorkflow |
| Add/delete node | Yes | Manual setWorkflow updater with undo stack push |
| Param change | Yes | Existing updateWorkflowParam pattern |

Positions persist through undo/redo — user's layout is not affected by undo of data changes.

---

## Handle Architecture (Critical Detail)

**Before:** Each node has one generic target handle and one generic source handle, no IDs
**After:** Target handles = one per input field with `id={fieldName}`; Source handles = one per output with `id={String(outputIdx)}`

Edge creation: `targetHandle: fieldName`, `sourceHandle: String(outputIdx)` — matches handle IDs

---

## Edge Case Handling

| Case | Solution |
|------|---------|
| Connection cleanup on node delete | Iterate all nodes, remove any `[deletedId, _]` tuples |
| Default value on disconnect | Reset to `''` — safe for all types |
| New node ID collision | `new_` prefix + incrementing counter + collision check |
| Saved positions re-parsing | layoutNodes applies saved positions first, grids the rest |
| New node placement | Viewport center (tracked via onMove) or `{x:400, y:300}` fallback |
| Unknown node types in I2V workflow | Default to column 2, default border color |
