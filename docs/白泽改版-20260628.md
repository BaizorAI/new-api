# 白泽改版计划 2026-06-28

## 核心判断

这次改版不是把第一阶段做好的左侧导航推倒重来，而是在第一阶段的导航结构上继续扩展。

第一阶段已经确定的主导航应继续保留，但最左边栏要拆成上下两块：

- 上对齐：日常工作入口，用户每天高频点击。
- 下对齐：管理、设置和账户入口，用户低频点击，主要服务运营、管理员和个人配置。

这样可以保留第一阶段低风险结构，同时让视觉重心更接近普通用户的工作路径。

第二阶段的重点是把“工作台”节点下的内容补完整，让普通用户一进来就能找到个人工作、团队协作、技能和成果，而不是进入一套新的侧栏或被带到技术后台。

## 设计原则

1. 保留第一阶段左侧导航作为主结构。
2. 第二阶段只扩展“工作台”节点，不替换整套侧栏。
3. `/team-workspace`、`/hermes-playground`、`/one-person-company` 这些工作区路由继续显示根导航。
4. 只有系统设置这类深层管理页面可以进入独立的上下文侧栏。
5. 团队工作台负责“协作做事”，团队管理负责“成员、角色、密钥、额度”等管理动作。
6. 最左边栏按“上方工作、下方管理”分区，避免普通用户入口和技术运维入口混在一起。

## 最左边栏分区

最左边栏只承担产品级入口，不承担深层对象列表。具体对象列表和上下文操作继续放在当前入口展开后的菜单里。

上对齐入口建议：

1. 工作台
   - 默认第一入口。
   - 承载工作台概览、爱马仕Agent、我的一人公司、团队工作台。
2. 技能商店
   - 找能力、复用技能、发布技能。
   - 面向“我要完成什么工作”，不是面向“我要管理一个配置”。
3. 消息平台
   - 微信、消息任务、连接状态。
   - 这是全局服务，不应藏在某个 Agent 页面下面。
4. 成果
   - 报告、PPT、文档、附件结果。
   - 用户完成任务后的主要回访入口。
5. 模型操场
   - 大模型试用和 AI 对话。
   - 保留给需要直接试模型的用户，但排在工作入口之后。

下对齐入口建议：

1. 管理
   - 团队管理、API 密钥、渠道、模型、用量日志、计费、用户、系统信息。
   - 面向管理员、运营和技术人员。
2. 设置
   - 个人资料、钱包、通知、安全、偏好。
   - 面向当前用户自己的配置。

不建议把“概览”作为最左边栏的独立入口长期保留。更合理的做法是把“工作台”作为默认入口，进入后第一项就是“工作台概览”。原因是“概览”本身不是一个产品能力，普通用户更容易理解“工作台”。

## 第一列到第二列映射计划

第二列不是新的功能集合，而是第一列当前入口下的对象列表和常用动作。每个第一列入口点击后，第二列应显示用户下一步最可能要点的内容。

| 第一列入口 | 第二列内容 | 首期实现方式 |
| --- | --- | --- |
| 工作台 | 工作台概览、爱马仕Agent、我的一人公司、团队工作台、团队列表、团队会话、团队成果、团队技能、团队成员、团队设置 | 复用现有工作台、爱马仕Agent、团队工作台和团队管理路由 |
| 技能商店 | 技能概览、我的技能、团队技能、白泽共享技能、内置技能、工具、技能分类 | 先统一进入爱马仕能力中心，后续在能力中心内补过滤和分类 |
| 消息平台 | 微信、执行任务、消息记录、连接设置 | 先进入爱马仕消息平台和执行任务面板，消息记录/连接设置作为同一入口下的二级导航 |
| 成果 | 全部成果、我的成果、团队成果、PPT、报告、文档、附件结果 | 先进入现有成果面板，后续在成果中心补类型和归属过滤 |
| 模型操场 | 模型会话、大模型试用、AI 对话 | 复用现有模型操场会话和 AI 对话预设 |
| 管理 | 团队管理、API 密钥、平台概览、数据看板、用量明细、任务记录、渠道、模型、用户、兑换码、订阅 | 保留现有管理路由，但放在下对齐区域 |
| 设置 | 钱包、个人资料、系统信息、系统设置 | 保留现有设置路由，放在下对齐区域 |

首期的重点是导航语义和入口分配正确，不强行一次性做完整筛选页。比如“我的技能 / 团队技能 / 白泽共享技能 / 内置技能”可以先都进入能力中心，后续再让能力中心按入口自动定位到对应分区。

## 工作台节点扩展

在左侧导航的“工作台”分组下保留并扩展这些入口：

- 工作台概览
  - 继续工作
  - 最近个人会话
  - 最近团队会话
  - 常用技能
  - 最新成果
  - 额度状态
- 爱马仕Agent
  - 个人 AI 工作台
  - 个人会话、技能、附件和成果
- 我的一人公司
  - 个人经营和项目工作台
  - 面向个人业务流程、项目推进和成果沉淀
- 团队工作台
  - 展示当前用户加入的团队
  - 每个团队下展示团队会话、团队成果、团队技能、团队成员、团队设置

团队节点的职责要分清：

- 团队会话：团队成员共享的对话和任务上下文。
- 团队成果：团队共享的报告、PPT、文档、附件结果。
- 团队技能：团队可复用的技能集合。
- 团队成员：进入团队成员视图。
- 团队设置：进入团队配置视图。

其中“团队成员”和“团队设置”可以链接到现有团队管理页面，但入口语义仍挂在具体团队下，方便用户理解。

## 技能商店

技能商店应从“技能管理”变成“找一个能力来完成工作”。

建议后续在技能商店下组织：

- 推荐
- 最近使用
- 我的技能
- 团队技能
- 白泽共享技能
- 内置技能
- 分类
  - PPT 和演示
  - 调研报告
  - 数据分析
  - 文档写作
  - 团队运营
  - 客户跟进

技能卡片优先表达：

- 能完成什么工作
- 可用于个人还是团队
- 是否来自白泽共享技能
- 是否可直接开始会话
- 是否可发布到团队

## 消息平台

消息平台是全局服务，不应只藏在爱马仕Agent下面。

第一期建议：

- 微信
- 执行任务
- 消息记录
- 连接设置

微信页面应展示：

- 当前连接状态
- 扫码连接
- 最近消息
- 最近任务
- 失败原因
- 重新连接

执行任务列表应展示：

- 正在执行
- 已完成
- 失败
- 可重新进入会话查看进度
- 可重试

这样能解决两个线上痛点：

- 微信消息偶尔没有反馈，用户不知道发生了什么。
- 离开聊天后任务中止，或无法重新进入查看进度。

## 成果中心

成果是普通用户非常关心的内容，应独立出来。

成果中心建议包含：

- 全部成果
- 我的成果
- 团队成果
- PPT
- 报告
- 文档
- 附件结果

成果卡片显示：

- 文件名
- 来源会话
- 所属团队
- 创建人
- 创建时间
- 下载、预览、复制链接
- 重新进入会话

## 管理区收口

技术和运营能力继续保留，但应放在管理或设置区域。

管理区建议包含：

- 团队管理
- API 密钥
- 渠道管理
- 模型管理
- 用量日志
- 计费管理
- 用户管理
- 系统信息

设置区建议包含：

- 个人资料
- 钱包
- 通知
- 安全
- 系统偏好

## 实施顺序

1. 保留第一阶段左侧导航，不再新增工作台专用侧栏替换视图。
2. 把最左边栏拆成上对齐和下对齐两块：上方放工作入口，下方放管理和设置。
3. 在“工作台”分组内补齐工作台概览、爱马仕Agent、我的一人公司、团队工作台。
4. 团队工作台下接入团队列表，并补齐团队会话、团队成果、团队技能、团队成员、团队设置。
5. 修复折叠状态下团队入口跳到裸 `/team-workspace` 造成的错误页问题。
6. 技能商店前置，先接入我的技能、团队技能、白泽共享技能、内置技能。
7. 消息平台独立出来，先接微信和执行任务。
8. 成果中心接入已有附件和生成文件。
9. 管理类入口统一收进“管理”，减少普通用户看到的技术术语。
10. 最后清理旧侧栏重复入口和不符合用户心智的术语。

## 当前阶段结论

第二阶段不是“把左侧导航改成 Coze 式双列结构”的大重构，而是先在第一阶段导航基础上扩展菜单。

Coze 的启发可以保留：普通用户先看到团队、技能、成果、消息。但实现上要走低风险路线，先把现有左侧导航和工作台节点做好，再评估是否需要真正的双列导航框架。
